# Complete API Specification

---

## Authentication APIs

```
POST /api/auth/register
  Body: { email, password, role, name, phone }
  Response: { token, user }

POST /api/auth/login
  Body: { email, password }
  Response: { token, user }

POST /api/auth/logout
  Response: { success: true }

POST /api/auth/verify
  Headers: { Authorization: "Bearer <token>" }
  Response: { user, valid: true }

GET /api/profile
  Description: Get current logged-in user details (reads JWT)
  Response: { user }

PUT /api/profile
  Description: Edit current user profile
  Body: { name?, phone?, companyDetails? }
  Response: { updatedUser }

DELETE /api/profile
  Description: Delete own account
  Response: { success: true } 


```

## APIs (Painter)

```

GET /api/jobs
  Description: Shows all the jobs for a painter.

GET /api/jobs/:jobId
  Description: Shows 1 job.

GET /api/jobs/:jobId/submissions
  Description: Shows all the submissions of a particular job:jobId by a painter.

GET /api/jobs/:jobId/submissions/:submissionId
  Description: Gives the detail of 1 particular submission by a painter.

PUT /api/jobs/:jobId/submissions/:submissionId
  Description: Edit a submission.

DELETE /api/jobs/:jobId/submissions/:submissionId
  Description: Delete a submission.

POST /api/jobs/:jobId/submissions
  Description: Create new painting submission by painter
  
```

## APIs (Owner)

```

POST /api/owners/:ownerId/jobs
  `create a new job .`

`API required for searching of painter when job creation occurs, also during job editing.`

PUT /api/owners/:ownerId/jobs
  `edit a job details (add painter, change name, etc)`

DELETE /api/owners/:ownerId/jobs
  `DELETE a job`

GET /api/owners/:ownerId/jobs
  `shows all the job created by the owner`

GET /api/owners/:ownerId/jobs/:jobId
  `enter a job to view.`
  
GET /api/owners/:ownerId/jobs/:jobId/files
  `show the files of a job:jobId`

GET /api/owners/:ownerId/jobs/:jobId/files/:fileId
  `show 1 file of a job:jobId`

GET /api/owners/:ownerId/jobs/:jobId/:painterId
  `will show all the submissions of a painter:painterId`

PUT /api/owners/:ownerId/jobs/:jobId/:painterId/:submissionId
  `edit the details of one submission`

`API required for the deletion of a photo under one submission.`

`API for the approval of submission.`

`API for rejection of submission.`

```

## File Generation APIs (can be generated by owner only)

```
POST /api/generate/excel
  Description: Trigger Excel file generation (for a jobId)

POST /api/generate/pdf-photo
  Description: Trigger watermarked photos PDF generation (for a jobId)

POST /api/generate/pdf-file
  Description: Trigger file PDF generation (for a jobId)

GET /api/generate/status/:generatedFileId
  Description: Check generation progress

```

## Photos APIs

```
GET /api/photos/:jobId
  Description: Get all photos for a job
  

GET /api/photos/:jobId/download-pdf
  Description: Download all watermarked photos as PDF (1 per page)


POST /api/photos/sign
  Description: Get Cloudinary upload signature

```

## File Management APIs

```
GET /api/excel/:fileId/download
  Description: Get download link for excel (R2 signed URL)
  
GET /api/files/:fileId/download
  Description: Get download link for pdf-files (R2 signed URL)

DELETE /api/files/:fileId
  Description: Delete generated file (excel, pdf-file or pdf-photo) (owner only)

```
